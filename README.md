📑 EL MANIFIESTO DEL ARQUITECTO: 100 CONCEPTOS DE ÉLITE (2026)
I. INFRAESTRUCTURA Y DESPLIEGUE (EL NÚCLEO WOZNIAK)
VPS (Virtual Private Server): Servidor particionado en la nube. Tu base de operaciones 24/7.

SSH (Secure Shell): Protocolo para entrar a tu servidor de forma segura mediante comandos.

Docker: Contenedores que empaquetan software para que corra igual en tu i3 que en la nube.

Docker Compose: Orquestador de varios contenedores (ej. n8n + base de datos).

Instance (Instancia): Una unidad de computación en la nube (ej. en Oracle Cloud).

ARM Architecture: Procesadores eficientes (como los de Oracle Free Tier) que dan más potencia por menos costo.

Latency (Latencia): El tiempo de respuesta. Baja latencia = Ejecución instantánea.

Uptime: El tiempo que tu sistema está vivo. La élite busca el 99.9%.

CLI (Command Line Interface): Tu terminal. Donde la velocidad supera al ratón.

Environment Variables (.env): Archivos secretos donde guardas llaves API sin exponerlas.

Root Access: Permiso total sobre el sistema. El poder absoluto del arquitecto.

Reverse Proxy: Servidor que dirige el tráfico (ej. Nginx) para que tu n8n sea seguro.

SSL/TLS: Cifrado que pone el candado verde en tu web/app. Vital para la confianza.

DNS Propagation: El tiempo que tarda el mundo en saber que tu dominio apunta a tu IP.

Localhost: Tu propia computadora actuando como servidor de prueba.

II. AUTOMATIZACIÓN E INGENIERÍA DE DATOS
Webhook: "Empuje" de datos en tiempo real. No preguntas, el dato te llega.

Polling: Lo opuesto al Webhook. Preguntar cada cierto tiempo (gasta recursos).

JSON (JavaScript Object Notation): El formato universal de intercambio de datos.

Array: Una lista de elementos (ej. una lista de correos de clientes).

Object: Un conjunto de propiedades (ej. Nombre, Email, Teléfono de un solo cliente).

Key-Value Pair: La estructura del dato. La "Llave" es el nombre, el "Valor" es el dato.

Mapping: El arte de conectar el "Dato A" con el "Campo B" sin romper la lógica.

Parsing: Traducir un bloque de texto rudo a datos organizados que la IA entienda.

Regex (Regular Expressions): Fórmulas para extraer texto específico (ej. un DNI en un PDF).

HTTP Request: Pedirle algo a un servidor (GET, POST, PUT, DELETE).

Status Code 200: Todo salió bien.

Status Code 401: No tienes permiso (lo que te pasó en Make).

Status Code 404: No se encontró la ruta.

Status Code 500: Error interno del servidor del proveedor.

Rate Limit: El límite de velocidad de una API para que no la satures.

Pagination: Cuando una API te da datos por "páginas" para no enviarte 10,000 de golpe.

Data Transformation: Cambiar el formato de un dato (ej. de fecha USA a fecha España).

Workflow: La secuencia lógica de pasos automatizados.

Nodes (Nodos): Cada paso individual en n8n o Make.

Binary Data: Archivos reales (imágenes, PDFs) viajando por tus flujos.

Token: La unidad de medida de las IAs. Un token ≈ 4 letras.

Zero-Shot Prompting: Pedirle algo a la IA sin darle ejemplos previos.

Few-Shot Prompting: Darle a la IA 2 o 3 ejemplos de cómo quieres que responda.

Chain of Thought: Técnica para que la IA "piense en voz alta" antes de decidir.

Vector Database: Base de datos especial donde la IA busca por "significado" (Pinecone, Weaviate).

Temperature: El nivel de "creatividad" de la IA. 0 = Lógico, 1 = Creativo.

System Prompt: Las instrucciones base que definen la personalidad de la IA.

Multimodalidad: IAs que procesan texto, audio, imagen y video simultáneamente.

Hallucination (Alucinación): Cuando la IA inventa datos con total seguridad.

FASE 1: INFRAESTRUCTURA Y HARDWARE (TU ESTUDIO TÉCNICO)
Conceptos necesarios para preparar tu Asus i3 y tu entorno local.

LTS (Long Term Support): Versión estable y recomendada de un software (como Node.js). Imprescindible para entornos profesionales.

Kernel: El núcleo del sistema operativo. Es el que gestiona el hardware de tu i3.

Node.js: El entorno que permite ejecutar JavaScript fuera del navegador. Tu "motor" local.

NPM (Node Package Manager): El almacén más grande de herramientas de software del mundo.

CLI (Command Line Interface): La terminal. Donde el experto escribe comandos en lugar de hacer clics.

Binary: Archivos compilados que la computadora ejecuta directamente.

PATH: La ruta que le dice a tu computadora dónde encontrar los programas instalados.

Shell: El intérprete de comandos (Bash en Linux, CMD/PowerShell en Windows).

CMD (Símbolo del Sistema): La terminal nativa de Windows que usarás hoy.

Environment: El "clima" de tu computadora; variables que definen cómo corre el software.

RAM (Random Access Memory): La memoria de trabajo. Vital para tener Make, Claude y GitHub abiertos.

CPU (Central Processing Unit): El cerebro de tu i3. Orquestador de cada proceso.

SSD: Disco de estado sólido. La velocidad de lectura que hace que tu entorno vuele.

Process: Una instancia de un programa en ejecución.

Localhost: La dirección IP interna de tu computadora (127.0.0.1). Tu servidor de pruebas.

Port (Puerto): La "puerta" virtual por la que entra y sale información (ej. Puerto 80 para web).

Cache: Memoria temporal que acelera procesos pero puede causar el error 401 si se corrompe.

Cookie: Fragmento de datos que identifica tu sesión (causante de conflictos de identidad).

IP Address: La matrícula de tu conexión a internet.

Static IP: Una dirección que nunca cambia, ideal para servidores profesionales.

Cloud Computing: Procesar datos en computadoras ajenas (Google, Oracle, AWS).

SaaS (Software as a Service): Software por suscripción, como Make o Canva.

PaaS (Platform as a Service): Entornos para construir apps sin gestionar el servidor (Heroku).

IaaS (Infrastructure as a Service): Servidores puros donde tú instalas todo (Oracle Cloud).

Virtualization: Crear una computadora "ficticia" dentro de una real.

FASE 2: IDENTIDAD DIGITAL Y CONTROL (TU FIRMA DE AUTORIDAD)
Conceptos para dominar GitHub y el registro de propiedad intelectual.

Git: El sistema que rastrea cada cambio en tu código. Tu seguro de vida técnico.

GitHub: La plataforma donde vive tu portafolio y colaboras con el mundo.

Repository (Repo): La "carpeta inteligente" donde vive tu proyecto.

Commit: Guardar un cambio oficial con un mensaje descriptivo. Tu "huella" histórica.

Push: Subir tus cambios locales a la nube de GitHub.

Pull: Bajar los cambios de la nube a tu computadora local.

Branch (Rama): Una línea paralela de trabajo para probar cosas sin romper el proyecto principal.

Merge: Fusionar una rama de prueba con la principal una vez que funciona.

Fork: Copiar el proyecto de otra persona para mejorarlo en tu propia cuenta.

Clone: Descargar un repositorio completo para trabajar en él.

Markdown (.md): El lenguaje sencillo para escribir documentación profesional (como este archivo).

README: El archivo de presentación de todo proyecto. Tu carta de ventas técnica.

Issue: Un reporte de error o una tarea pendiente dentro de GitHub.

Pull Request (PR): Pedir permiso para integrar tus cambios en un proyecto ajeno.

Gitignore: Archivo que le dice a Git qué carpetas NO debe subir (como tus contraseñas).

SSH Key: Una llave digital ultra segura para conectar tu i3 con GitHub sin usar contraseñas.

Personal Access Token (PAT): Una llave temporal para dar permisos a apps externas.

CI/CD: Automatización que prueba y sube tu código solo si no tiene errores.

Deployment: El acto de poner tu solución a funcionar en el mundo real.

Staging: Un entorno de "ensayo" idéntico al real pero privado.

Production (Prod): El entorno real donde los clientes usan tu solución.

Version Control: La disciplina de gestionar versiones para nunca decir "perdí el archivo".

Open Source: Software cuyo código es público y puede ser mejorado por cualquiera.

Private Repo: Repositorio oculto para tus soluciones "Secretas" de pago.

Organization: Una cuenta grupal en GitHub para tu futura agencia.

FASE 3: LÓGICA DE FLUJO Y AUTOMATIZACIÓN (TU LABORATORIO)
Conceptos clave para dominar Make y n8n (Semana 1).

Workflow: El mapa completo de una automatización.

Scenario: El nombre que Make le da a un flujo de trabajo.

Module (Módulo): Cada aplicación o herramienta dentro de Make.

Node (Nodo): El equivalente a módulo en n8n.

Trigger (Disparador): El evento que inicia todo (ej. "Llegó un email").

Action (Acción): Lo que el sistema hace después del trigger.

Webhook: Un "oído" digital que recibe datos al instante. La base de la velocidad.

Polling: Revisar cada X minutos si hay algo nuevo. Lento y costoso.

Router: El divisor de caminos. Permite tomar decisiones lógicas (SI/NO).

Filter: Un portero que solo deja pasar los datos que cumplen una regla.

Iterator: Herramienta para procesar una lista de elementos uno por uno.

Aggregator: Lo opuesto al iterator; junta muchos elementos en una sola lista.

Data Store: Una base de datos interna y rápida dentro de Make.

Variable: Un espacio para guardar un dato que cambia (ej. el precio de hoy).

Global Variable: Un dato accesible desde cualquier parte de tu cuenta.

Operation (Operación): La unidad de medida de gasto en Make. Cada paso gasta una.

Task (Tarea): El equivalente a operación en otras plataformas.

Execution: El acto de correr un flujo de principio a fin.

History: El registro de qué pasó en cada ejecución pasada.

Blueprint: El plano técnico (JSON) de tu automatización que puedes vender.

Error Handler: Una ruta de emergencia por si algo falla (ej. "Avisar por WhatsApp").

Break: Detener el flujo por completo ante un error crítico.

Resume: Intentar continuar el flujo después de corregir un error.

Ignore: Seguir adelante aunque un paso haya fallado (peligroso).

Commit (en DB): Confirmar que los datos se escribieron permanentemente.

FASE 4: INGENIERÍA DE DATOS Y APIS (EL IDIOMA DEL WOZNIAK)
Conceptos para conectar IAs y software complejo (Semana 2).

API (Application Programming Interface): El puente de comunicación entre softwares.

Endpoint: La dirección específica de una API (ej. api.google.com/gemini).

JSON (JavaScript Object Notation): El formato de texto universal para enviar datos.

Key-Value Pair: Estructura básica: "Nombre": "Carlos".

Array: Una lista de datos: ["Manzana", "Pera", "Uva"].

Object: Un conjunto de datos de una sola entidad: {Nombre: "Carlos", Edad: 30}.

Parsing: El proceso de convertir un texto JSON en datos que Make pueda usar.

String: Cualquier dato que sea texto puro.

Integer: Un número entero (sin decimales).

Boolean: Un dato que solo puede ser Verdadero (True) o Falso (False).

Null: La ausencia total de valor. No es cero, es "vacío".

Payload: El contenido real de datos que envías en una petición.

Header: Información "oculta" en una petición (como tu API Key).

Authentication (Auth): El proceso de demostrar quién eres.

Bearer Token: Una llave de acceso larga y segura que viaja en el Header.

API Key: Tu contraseña para usar un servicio (ej. Gemini). Nunca se comparte.

Rate Limit: El límite de velocidad. "No puedes pedir más de 10 cosas por segundo".

Timeout: Cuando una petición tarda demasiado y el sistema se rinde.

HTTP Status Codes: El lenguaje de respuesta del servidor (200=OK, 401=Error).

GET: Pedir información a un servidor.

POST: Enviar información nueva a un servidor.

PUT: Reemplazar información existente por una nueva.

DELETE: Borrar información del servidor.

PATCH: Editar solo una parte pequeña de una información.

Encoding: El formato del texto (ej. UTF-8 para que las tildes se vean bien).

FASE 5: INTELIGENCIA ARTIFICIAL (EL CEREBRO DE TUS SOLUCIONES)
Conceptos para dominar Gemini, Claude y la lógica de agentes.

LLM (Large Language Model): El motor de IA (Gemini, GPT, Claude).

Token: La unidad mínima de procesamiento. Unas 4 letras.

Prompt: La instrucción que le das a la IA. Tu comando de voz.

System Prompt: Las reglas de comportamiento "profundas" de la IA.

Zero-Shot: Pedir algo sin dar ejemplos.

Few-Shot: Dar 2 o 3 ejemplos para que la IA entienda el formato.

Hallucination: Cuando la IA inventa algo con mucha seguridad.

Temperature: Control de creatividad. 0 es robótico, 1 es poeta.

Context Window: Cuánta información puede "recordar" la IA en una sola charla.

RAG (Retrieval-Augmented Generation): Conectar la IA a tus PDFs o bases de datos.

Vector Database: Base de datos que busca por significado, no por palabra exacta.

Embedding: Convertir una frase en una coordenada numérica (vector).

Multimodal: IA que puede ver imágenes, escuchar audios y leer texto a la vez.

Agent: Una IA que tiene autonomía para usar herramientas (ej. "Busca en Google").

Function Calling: Cuando la IA decide qué herramienta usar por sí sola.

Reasoning: La capacidad de la IA de explicar su paso a paso lógico.

Perplexity: Una IA especializada en buscar en internet en tiempo real.

Sonnet/Opus/Haiku: Los nombres de las versiones de Claude (Anthropic).

Gemini Flash vs Pro: Flash es rápida y barata; Pro es inteligente y profunda.

DeepSeek: Modelo de IA ultra eficiente que está rompiendo el mercado.

Prompt Injection: Un ataque donde alguien intenta engañar a tu IA.

Sentiment Analysis: Usar IA para saber si un cliente está feliz o enfadado.

Summarization: Resumir textos largos automáticamente.

Classification: Etiquetar datos (ej. "Este correo es una QUEJA").

Translation: Cambiar el idioma manteniendo el contexto cultural.

Lead: Un cliente potencial que te ha dejado sus datos.

BLOQUE I: INFRAESTRUCTURA LOCAL Y SISTEMA OPERATIVO (SEMANA 1 - INICIO)
El dominio del Wozniak sobre su propia máquina.

Kernel: El núcleo que comunica el software con el hardware de tu i3.

CPU Threads: Hilos de procesamiento. Vital para entender la concurrencia en tareas pesadas.

RAM Swapping: Cuando el PC usa el disco como RAM. Evítalo para no ralentizar el desarrollo.

LTS (Long Term Support): Versiones de software con soporte extendido (ej. Node.js 20.x).

Runtime Environment: El entorno donde se ejecuta el código (ej. Node.js).

Package.json: El archivo "ADN" de cualquier proyecto de Node.js que lista sus dependencias.

Node_modules: La carpeta donde viven las herramientas que descargas vía NPM.

Dependency Hell: Conflicto de versiones entre librerías. Se soluciona con package-lock.json.

Environment Variables (.env): Archivos para guardar API Keys sin que se vean en el código.

PATH Variable: La lista de rutas donde el sistema busca comandos ejecutables.

Bash / PowerShell: Los lenguajes de la terminal para automatizar el sistema operativo.

CLI Agent: IA que vive en tu terminal y tiene permiso para tocar archivos (ej. Claude Code).

SSH (Secure Shell): El túnel cifrado para controlar servidores remotos.

SSH Key Pair: Llave pública (candado) y privada (llave) para entrar a servidores sin clave.

Sudo / Administrator: Permisos de superusuario. Úsalos con cautela.

Port Forwarding: Abrir un túnel para que un servicio de tu i3 sea visible en internet.

DNS (Domain Name System): El listín telefónico que traduce google.com a una IP.

A Record: El registro DNS que apunta un dominio directamente a una IP.

CNAME: Un alias de un dominio a otro dominio.

TTL (Time To Live): El tiempo que tarda un cambio de DNS en propagarse por el mundo.

IP Static vs Dynamic: Estática nunca cambia (servidores); Dinámica cambia (tu casa).

IPv6: El nuevo estándar de direcciones IP ante el agotamiento de IPv4.

Latency: El retraso en milisegundos de una señal. Crítico en automatizaciones en tiempo real.

Throughput: La cantidad de datos que tu sistema puede procesar por segundo.

Bandwidth: El ancho de banda disponible para mover datos.

Virtualization: Ejecutar un sistema operativo dentro de otro (ej. WSL2 en Windows).

WSL2 (Windows Subsystem for Linux): Tener Linux dentro de tu Windows para un entorno PRO.

Distro (Distribución): Versión de Linux (ej. Ubuntu, Debian).

GUI (Graphical User Interface): Lo que ves con el ratón. El experto la usa poco.

Headless: Software que corre sin interfaz gráfica, ahorrando toda la RAM.

Daemon: Un proceso que corre en segundo plano permanentemente.

Cron Job: Una tarea programada para ejecutarse a una hora exacta en el sistema.

Binary Data: Datos en formato 0 y 1 (imágenes, audios).

Encoding (UTF-8): El estándar que permite que las tildes y eñes no se rompan.

Base64: Formato para enviar imágenes o archivos como si fueran simple texto.

Local Storage: Almacenamiento persistente en el navegador.

Session Storage: Almacenamiento que se borra al cerrar la pestaña.

Cache Invalidation: Borrar la memoria temporal para que se carguen los cambios nuevos.

Hot Reload: Cuando el software se actualiza solo en cuanto guardas un cambio de código.

Depuración (Debugging): El arte de encontrar por qué el sistema no hace lo que quieres.

BLOQUE IV: INGENIERÍA DE APIS Y ESTRUCTURA DE DATOS (SEMANA 2 - INICIO)
El idioma universal del intercambio de valor.

API (Application Programming Interface): El contrato de comunicación.

REST (Representational State Transfer): El estándar más usado de APIs.

Endpoint: La URL exacta a la que llamas (ej. /v1/chat/completions).

HTTP Methods: Las acciones (GET, POST, PUT, DELETE, PATCH).

GET Request: Pedir información (No tiene "body").

POST Request: Crear algo nuevo (Envía datos en el "body").

PUT vs PATCH: PUT reemplaza todo; PATCH solo edita un trocito.

HTTP Headers: Metadatos (ej. "Soy Carlos", "Uso JSON", "Mi clave es XXX").

Content-Type: Decirle al servidor qué le envías (application/json).

User-Agent: Identificar qué programa está haciendo la llamada.

Authorization Header: Donde pones tu Bearer Token o API Key.

Query Parameters: Datos en la URL (ej. ?search=zapatos&limit=10).

Path Parameters: Datos dentro de la ruta (ej. /users/123).

Response Body: Lo que el servidor te devuelve (normalmente un JSON).

Status Codes (2xx): Éxito (200=OK, 201=Creado).

Status Codes (4xx): Tu culpa (401=No autorizado, 404=No encontrado).

Status Codes (5xx): Culpa de ellos (500=Servidor roto, 503=Mantenimiento).

JSON Object: { "llave": "valor" }.

JSON Array: [ "item1", "item2" ].

Nested Data: Objetos dentro de objetos (ej. Cliente -> Dirección -> Ciudad).

Root Element: El nivel más alto de un JSON.

DataType (String): Texto.

DataType (Integer/Float): Números enteros / decimales.

DataType (Boolean): true / false.

DataType (Null): Valor vacío, ausencia de dato.

XML: El abuelo del JSON. Más pesado y difícil de leer.

CSV (Comma Separated Values): Formato para tablas y Excel.

API Documentation: El manual de instrucciones de una API (ej. Swagger).

Postman: La herramienta reina para probar APIs antes de llevarlas a Make.

Bearer Token: Una llave temporal y segura que caduca.

OAuth2: Sistema complejo para loguearse "como Google" o "como Facebook".

API Secret: La parte de la llave que NUNCA debe viajar en la URL.

CORS: Error de seguridad cuando una web intenta llamar a una API que no le pertenece.

Websocket: Conexión permanente. Los datos fluyen en ambos sentidos sin parar.

Payload: La carga útil. El mensaje real que envías.

El idioma del intercambio de valor. Aquí dejas de depender de los conectores prefabricados y empiezas a hablar directamente con los servidores.

46. API (Application Programming Interface)
Cero: Un "enchufe" que permite que dos programas diferentes hablen entre sí.

Wozniak: Un conjunto de reglas y protocolos que un servidor expone para que otros sistemas (como tu computadora o Make) puedan interactuar con su base de datos de forma segura, sin tener acceso al código fuente.

Jobs: El puente comercial. Si un cliente quiere conectar su anticuado sistema de contabilidad con WhatsApp, tú no tienes que reprogramar el sistema; solo buscas si tiene una API, te conectas y cobras por el puente.

Objetivo: Buscar en Google "Documentación API Stripe" y ver cómo luce un manual técnico real.

47. Endpoint
Cero: La dirección web exacta a la que le pides o envías información.

Wozniak: Una URL específica (ej. api.empresa.com/v1/usuarios) que expone una función particular de la API. Cada Endpoint está diseñado para recibir un tipo específico de petición y devolver una respuesta estructurada.

Jobs: Tu catálogo de servicios por plataforma. Conocer el Endpoint de "Crear Factura" te permite automatizar la contabilidad, mientras que el Endpoint de "Añadir Contacto" te permite alimentar el CRM.

Objetivo: Identificar la estructura de un Endpoint típico (con barras / separando categorías).

48. HTTP Request (Petición HTTP)
Cero: El "mensaje" completo que le mandas a una página web o servidor.

Wozniak: El paquete de comunicación base de internet. Se compone de un Método (verbo), una URL (Endpoint), Headers (metadatos) y un Body (datos). Make tiene un módulo llamado "Make an API Call" que hace esto de forma manual.

Jobs: Tu llave maestra. El 90% de los "expertos" de Make se rinden si no hay un módulo oficial para la app del cliente. El 0.5% usa un módulo HTTP Request, construye la petición a mano y cierra un contrato que los demás no pudieron.

Objetivo: Añadir el módulo "HTTP Request" en Make y ver sus campos en blanco.

49. GET Request
Cero: La orden de "Tráeme esta información".

Wozniak: El método HTTP diseñado exclusivamente para solicitar y recuperar datos de un servidor sin modificarlos. Viaja ligero, generalmente sin "Body", pasando los parámetros directamente en la URL.

Jobs: Lectura de datos. Es lo que usas para que tu automatización le pregunte al banco del cliente: "¿El pago #456 ya entró?" antes de enviar el producto.

Objetivo: Entender que cuando escribes google.com en tu navegador y das Enter, tu i3 está haciendo un GET Request.

50. POST Request
Cero: La orden de "Crea o guarda esta nueva información".

Wozniak: El método HTTP para enviar datos al servidor (dentro del Body) con la intención de crear un nuevo registro en la base de datos (como un nuevo usuario o un nuevo mensaje).

Jobs: Ejecución de ventas. Es el método que usas para enviar un lead recién captado desde Facebook directamente a la base de datos del cliente (CRM).

Objetivo: Saber que el POST siempre necesita un "Payload" (el paquete de datos que vas a entregar).

51. HTTP Headers (Cabeceras)
Cero: La información "oculta" que va en el sobre de la carta antes de leer el mensaje.

Wozniak: Pares clave-valor que acompañan a la petición HTTP y dan contexto al servidor (ej. Content-Type: application/json). Aquí es donde viajan las llaves de seguridad para demostrar quién eres.

Jobs: El control de aduanas. Configurar mal los Headers significa que el servidor del cliente rechazará tu conexión. Dominarlos te asegura acceso a cualquier sistema corporativo.

Objetivo: Aprender que la autorización siempre viaja en el Header, nunca en la URL.

52. Payload / Body (Cuerpo de la Petición)
Cero: El contenido real del mensaje o los datos que estás enviando.

Wozniak: La carga útil. Generalmente escrita en formato JSON, contiene los datos exactos que el servidor necesita procesar en una petición POST o PUT (ej. {"nombre": "Carlos", "edad": 30}).

Jobs: La información del cliente. Si el Body está mal estructurado, el CRM del cliente guardará los apellidos en el campo del teléfono y el sistema colapsará.

Objetivo: Escribir tu primer bloque de texto JSON en un bloc de notas.

53. Status Codes (Códigos de Estado HTTP)
Cero: El número que te devuelve el servidor para decirte si todo fue bien o mal.

Wozniak: La respuesta estandarizada de tres dígitos. Los 2xx significan éxito (200 OK, 201 Creado). Los 4xx significan que tú (el cliente) te equivocaste (400 Bad Request, 401 Unauthorized). Los 5xx significan que el servidor de ellos se rompió.

Jobs: Diagnóstico instantáneo en llamadas de soporte. Si una automatización falla y Make te muestra un Error 401, le dices al cliente en 3 segundos: "Tu contraseña de la API caducó", demostrando un control total.

Objetivo: Memorizar que el 200 es "Perfecto" y el 401 es "Acceso Denegado".

54. API Key / Bearer Token
Cero: Tu contraseña secreta para usar la app de otra empresa.

Wozniak: Cadenas de texto criptográficas que el servidor exige en los Headers para autenticar la petición. El Bearer Token es temporal y la API Key suele ser permanente (y muy peligrosa si se filtra).

Jobs: La llave de la caja fuerte. Si le pides la API Key de Stripe a un cliente y se la pasas por un chat inseguro, eres un riesgo. Un arquitecto maneja estas llaves con extrema seguridad.

Objetivo: Generar tu primera API Key en Google AI Studio (para usar a Gemini).

55. JSON Object (Objeto JSON)
Cero: Una "caja" que contiene información sobre una sola cosa, encerrada en llaves { }.

Wozniak: La estructura fundamental de datos. Representa una entidad individual con sus propiedades encapsuladas (ej. { "cliente": "Juan", "estado": "Activo" }).

Jobs: Organización de perfiles. Un Objeto JSON es el equivalente a la ficha completa de un cliente en tu sistema de facturación.

Objetivo: Escribir un Objeto JSON simple con tu nombre y tu especialidad.

56. JSON Array (Lista JSON)
Cero: Una lista que contiene varios Objetos o datos, encerrada en corchetes [ ].

Wozniak: Una colección ordenada de elementos. Puede contener cadenas de texto ["A", "B"] o múltiples objetos [ {id: 1}, {id: 2} ]. Requiere un módulo Iterator en Make para procesarse uno a uno.

Jobs: Manejo de volumen. Es el formato en el que te entregan 5.000 clientes a la vez. Saber procesar un Array significa que puedes facturar por limpiar bases de datos masivas.

Objetivo: Diferenciar visualmente las llaves { } (una cosa) de los corchetes [ ] (muchas cosas).

57. Key-Value Pair (Par Clave-Valor)
Cero: La regla de oro: Cada dato tiene una "Etiqueta" y un "Contenido".

Wozniak: La sintaxis base de un JSON. La clave siempre es un String entre comillas (ej. "email"), seguida de dos puntos :, y el valor puede ser texto, número, booleano u otro objeto (ej. "carlos@gmail.com").

Jobs: El mapeo perfecto. Cuando conectas sistemas, tienes que asegurar que la Key "correo" de una app se conecte con la Key "email" de la otra. Si no, se pierde el cliente.

Objetivo: Aprender que la clave siempre va a la izquierda de los dos puntos.

58. Parsing (Parseo de Datos)
Cero: Traducir un bloque de texto que parece basura en datos ordenados que Make puede usar.

Wozniak: El proceso de analizar una cadena de texto (String) y convertirla en una estructura de datos real en memoria (como un objeto JSON). En Make, esto se hace con el módulo "JSON Parse".

Jobs: Minería de oro. Un cliente te da un bloque de texto gigante lleno de facturas desordenadas; tú usas Inteligencia Artificial y un módulo de Parseo para convertirlo en un Excel perfecto en 10 segundos.

Objetivo: Entender que si Make ve el texto como "un solo bloque gris", necesitas Parsearlo para ver sus variables de colores separadas.

59. Stringify (Serialización)
Cero: El proceso inverso al Parseo. Convertir datos organizados en un bloque de texto plano.

Wozniak: Transformar un objeto JSON en una cadena de texto String para poder enviarlo de forma segura a través de internet (en el Body de un POST Request).

Jobs: Empaquetar la entrega. Es como cerrar la caja y ponerle cinta antes de dársela al mensajero (la red HTTP) para que nada se caiga en el camino.

Objetivo: Saber que cuando envías datos por un Webhook, Make los "Stringifica" automáticamente por ti.

60. Rate Limiting (Límite de Peticiones)
Cero: El límite de velocidad de la plataforma. "No puedes pedir más de 10 cosas por segundo".

Wozniak: Una medida defensiva de las APIs (Error 429 Too Many Requests) para evitar ataques o sobrecargas. Requiere implementar módulos de "Sleep" (Pausa) o manejo de errores en Make para no bloquear el escenario.

Jobs: Gestión de crisis. Si tu campaña publicitaria es un éxito y entran 1.000 leads de golpe, un sistema sin gestión de Rate Limit colapsará y perderás las ventas. El arquitecto previene esto pausando el tráfico.

Objetivo: Preguntarte siempre "¿Cuántas peticiones por minuto soporta esta app?" antes de construir un escenario.

BLOQUE V: INTELIGENCIA ARTIFICIAL Y LLMs (SEMANA 5)

61. LLM (Large Language Model)
Cero: El cerebro artificial (como ChatGPT, Claude o Gemini) que lee y escribe como un humano experto.

Wozniak: Una red neuronal basada en la arquitectura Transformer, entrenada con terabytes de datos para predecir el token estadísticamente más probable. No tiene consciencia; hace cálculos probabilísticos a velocidad luz.

Jobs: El "empleado infinito". Un activo que cuesta céntimos por tarea, no duerme, no se queja, no pide vacaciones y puede procesar 1.000 correos de atención al cliente en un solo minuto.

Objetivo: Entender que la IA es una herramienta de cálculo estadístico avanzado, no un humano.

62. Token
Cero: La "moneda" o unidad de medida que las IAs usan para leer y escribir (aprox. 4 letras de una palabra equivalen a 1 token).

Wozniak: La fragmentación de texto en unidades sub-léxicas que el modelo vectoriza. El algoritmo procesa y cobra por tokens, no por palabras completas.

Jobs: Tu estructura de costes base. Las APIs de Google (Gemini) o Anthropic (Claude) te cobran por millón de tokens procesados. Optimizar tus envíos significa gastar menos y aumentar el margen de beneficio de tu agencia.

Objetivo: Saber calcular mentalmente que un texto de 100 palabras son aproximadamente 130 tokens.

63. Prompt Engineering (Ingeniería de Instrucciones)
Cero: El arte de darle órdenes tan exactas a la IA que le sea imposible equivocarse.

Wozniak: El diseño técnico de inputs usando delimitadores (como """ o etiquetas XML <dato>), roles, variables de contexto y restricciones de salida (forzando formato JSON).

Jobs: Tu Propiedad Intelectual más valiosa. Un prompt que clasifica leads a la perfección es un activo digital por el que una empresa pagará miles de euros. En el 0.5%, no vendes "acceso a la IA", vendes tu Prompt perfecto.

Objetivo: Dejar de tratar a la IA como un buscador de Google y empezar a programarla usando lenguaje natural estructurado.

64. System Prompt (Instrucción de Sistema)
Cero: El "papel" o personaje base que la IA debe actuar (ej. "Eres un vendedor experto y amable").

Wozniak: El contexto de nivel raíz que condiciona todas las respuestas en la ventana de contexto. Se procesa con mayor peso estadístico que las órdenes del usuario para evitar desvíos.

Jobs: El ADN del negocio. Si el System Prompt es débil, el bot del cliente terminará hablando de política con un usuario molesto. Un buen System Prompt blinda la reputación de la empresa.

Objetivo: Redactar un System Prompt férreo que defina el tono de voz, los límites éticos y el objetivo del bot.

65. User Prompt (Instrucción de Usuario)
Cero: La tarea específica o el dato que le mandas a la IA en ese instante exacto.

Wozniak: El input dinámico (la variable) que entra a la API en cada ejecución de Make. Por ejemplo, el cuerpo del correo recién recibido que queremos que la IA analice.

Jobs: El factor de riesgo humano. Debes garantizar que sin importar qué locura escriba el cliente (el User Prompt), las reglas de tu sistema (System Prompt) controlen la respuesta final.

Objetivo: Separar claramente en tu mente qué texto es "tu regla fija" y qué texto es "el dato variable del cliente".

66. Context Window (Ventana de Contexto)
Cero: La memoria a corto plazo de la IA. Cuántos libros puede leer a la vez antes de "olvidar" el principio.

Wozniak: El límite máximo de tokens que un modelo puede procesar en una sola llamada a la API (Input + Output). Gemini 1.5 Pro domina esto en el mercado con hasta 2 millones de tokens.

Jobs: Análisis masivo a escala. Una ventana gigante te permite subirle a la IA todo el historial contable de 5 años de un cliente de una sola vez, para que extraiga patrones financieros.

Objetivo: Entender por qué un chatbot estándar "olvida" de qué hablaban después de 10 mensajes.

67. Temperature (Temperatura)

Wozniak: El hiperparámetro que ajusta la aletoriedad en la selección del siguiente token. Temperatura 0 = la IA elige siempre la opción estadística más segura (perfecta para código y JSON). Temperatura 1 = opciones diversas y creativas (poesía, marketing).

Jobs: Garantía de calidad de datos. Si extraes facturas automatizadas y no pones la temperatura en 0, la IA se pondrá "creativa" e inventará un número de cuenta falso.

Objetivo: Configurar siempre la Temperatura a 0.0 cuando necesites que la IA procese procesos lógicos o bases de datos en Make.

69. Zero-Shot Prompting
Cero: Pedirle a la IA que haga una tarea sin darle ningún ejemplo previo.

Wozniak: Explotar la capacidad de inferencia pura del modelo base. Consume menos tokens (más barato) y es más rápido, pero tiene mayor margen de error estructural.

Jobs: Tu herramienta para tareas rápidas y directas (ej. resumir un texto largo). Ahorra gastos de servidor.

Objetivo: Usar Zero-Shot solo cuando el formato de la respuesta no sea vital para el siguiente paso del proceso.

70. Few-Shot Prompting
Cero: Darle a la IA 2 o 3 ejemplos de un trabajo perfecto antes de pedirle que lo haga ella.

Jobs: Auditoría y transparencia. Si el bot califica a un lead como "basura", el dueño de la empresa querrá saber por qué. CoT te permite guardar el razonamiento secreto de la IA en tu base de datos para justificar la decisión.

Jobs: El argumento de ventas imbatible. Cuando le demuestras a un CEO (con un Excel) que pagarte 2.000€ le generará 15.000€ en ahorros de personal, vender es un trámite, no un esfuerzo.

91. VPS (Virtual Private Server)
Cero: Una computadora de alquiler que vive en internet, no tiene pantalla y nunca se apaga.

Wozniak: Una máquina virtual particionada dentro de un servidor físico. Te da acceso root (total) para instalar Linux, Node.js y plataformas como n8n sin las severas restricciones de un alojamiento web (hosting) compartido.

Jobs: Tu fábrica de producción 24/7. Alojar tus propias automatizaciones aquí reduce tus costos operativos a casi cero (comparado con pagar suscripciones caras), multiplicando el margen de ganancia que le cobras al cliente.

Objetivo: Entender que un VPS es tu "oficina en la nube" personal.

92. Proveedores Cloud (Oracle / AWS / DigitalOcean)
Cero: Las empresas gigantes que son dueñas de los edificios llenos de computadoras que alquilamos.

Wozniak: Proveedores de Infraestructura como Servicio (IaaS). Por ejemplo, Oracle Cloud ofrece instancias con arquitectura ARM (muy potentes y gratuitas en su capa Free Tier), perfectas para desplegar contenedores.

Jobs: Tus socios de cimientos. Elegir el proveedor correcto significa que puedes tener el "cerebro" de tu agencia corriendo gratis o por solo 5€ al mes, garantizando escalabilidad a largo plazo.

Objetivo: Saber que buscarás la "Capa Gratuita" (Free Tier) de Oracle o AWS cuando montes tu primer servidor real.

93. Docker
Cero: Cajas de envío mágicas. Metes un programa ahí, y funciona igual y sin fallos en cualquier computadora del mundo.

Wozniak: Plataforma de contenerización. Empaqueta tu código, las librerías necesarias y el entorno de ejecución en un paquete estándar e inmutable. Elimina la excusa de "en mi computadora sí funcionaba".

Jobs: Escalabilidad instantánea. Si un cliente crece de 100 a 100.000 clientes, con Docker clonas el sistema a servidores más grandes en 3 minutos. Es infraestructura corporativa.

Objetivo: Instalarás Docker Engine en tu futuro VPS para que n8n corra de forma segura.

94. Contenedor (Container) vs Imagen (Image)
Cero: La "Imagen" es el plano arquitectónico; el "Contenedor" es el edificio ya construido y vivo.

Wozniak: Una imagen es un archivo de solo lectura con las instrucciones (ej. la imagen oficial de n8n). Un contenedor es una instancia aislada y en ejecución de esa imagen. Comparten el kernel (núcleo) de la máquina host, siendo ultra ligeros.

Jobs: Eficiencia extrema. Puedes tener el CRM, el automatizador y la base de datos corriendo en el mismo servidor barato de 5€ sin que se "estorben" ni se corrompan entre sí.

Objetivo: Entender que si un contenedor se "rompe", lo borras y creas otro idéntico a partir de la imagen en 5 segundos.

95. Docker Compose
Cero: El director de orquesta que enciende varias cajas mágicas al mismo tiempo y las conecta por cables invisibles.

Wozniak: Una herramienta (mediante un archivo docker-compose.yml) para definir y ejecutar aplicaciones Docker multi-contenedor. Con un solo comando, levantas la aplicación y su base de datos PostgreSQL conectadas por una red interna.

Jobs: Despliegues en un clic. Te permite instalar infraestructuras complejas para clientes en 10 segundos. Cobras por el inmenso valor del sistema, pero te cuesta literalmente un comando de instalación.

Objetivo: Leerás un archivo .yml y verás cómo estructura los servicios de tu infraestructura.

96. SSH (Secure Shell)
Cero: Un túnel secreto y blindado para controlar tu computadora de la nube (VPS) desde la pantalla negra de tu i3.

Wozniak: Protocolo de red criptográfico para operar servicios remotamente. A través de la terminal, te conectas al puerto 22 del servidor usando tus claves privadas para administrar el sistema operativo Linux.

Jobs: El acceso de los verdaderos expertos. Un cliente confía más en un consultor que administra servidores desde una terminal negra que en uno que necesita paneles visuales infantiles.

Objetivo: Dominar el comando de conexión básica: ssh usuario@ip_del_servidor.

97. Reverse Proxy (Nginx / Traefik)
Cero: El portero del edificio. Recibe a la gente de internet y la dirige a la puerta de la habitación correcta dentro de tu servidor.

Wozniak: Un servidor que se sitúa frente a tus aplicaciones web y reenvía las solicitudes (HTTP) de los usuarios hacia el contenedor Docker correspondiente. Maneja el tráfico de red y balancea la carga.

Jobs: Tu imagen corporativa. Permite que el cliente entre a un sistema elegante como crm.tuempresa.com en lugar de una IP horrible y no profesional como 192.168.1.5:5678.

98. SSL / TLS (Let's Encrypt)
Cero: El famoso candado verde que sale arriba en el navegador. Significa que los datos están cifrados.

Wozniak: Protocolos criptográficos que aseguran las comunicaciones por internet. Let's Encrypt es la autoridad que automatiza la entrega de certificados gratuitos. Sin esto, todo dato que viajes (contraseñas, JSONs) puede ser interceptado.

Jobs: Confianza innegociable. Ningún cliente meterá su tarjeta ni sus datos en una web que el navegador marque como "No seguro". Sin SSL, no hay embudo de ventas que funcione.

Objetivo: Asegurar que TODAS las URLs de tus Webhooks empiecen siempre con https:// y no con http://.

99. DNS (Domain Name System)
Cero: El directorio telefónico de internet. Traduce el nombre google.com al número de la computadora real de Google.

Wozniak: El sistema jerárquico que traduce nombres legibles para humanos a direcciones IP numéricas que las redes usan para enrutar el tráfico mundial.

Jobs: Tu marca y posicionamiento. Comprar y configurar correctamente un dominio (DNS) es el primer paso indispensable para establecer la presencia de "CarlosHernandezVasco-solutions".

Objetivo: Entrar a un proveedor de dominios (como Namecheap o Cloudflare) y entender dónde está el panel de configuración DNS.

100. A Record & CNAME (Registros DNS)
Cero: Las flechas de dirección. Le dicen al dominio a qué computadora exacta apuntar.

Wozniak: El registro 'A' (Address) apunta tu dominio directamente a la IP pública de tu servidor. El 'CNAME' (Canonical Name) apunta un subdominio (ej. www o api) a otro dominio existente.

Jobs: Control de crisis. Si el servidor de un cliente se incendia, un Arquitecto experto cambia el registro 'A' hacia un servidor de respaldo en 5 minutos, salvando el día y su facturación.

Objetivo: Sabrás que para conectar tu propio dominio a tus automatizaciones necesitas crear un Registro 'A' hacia tu VPS.

101. Firewall (Cortafuegos / UFW)
Cero: El muro de fuego que bloquea a los atacantes para que no entren a tu computadora en la nube.

Wozniak: Sistema de seguridad de red que monitorea el tráfico basado en reglas estrictas. En Linux usamos UFW para cerrar absolutamente todos los puertos y abrir solo el 80 (HTTP), 443 (HTTPS) y 22 (SSH).

Jobs: Tu seguro anti-demandas. Si un hacker entra al servidor de tu cliente porque dejaste los puertos de la base de datos abiertos, la responsabilidad legal es tuya. El firewall es tu abogado técnico.

Objetivo: Entender la regla de oro: Por defecto, todo en el servidor está cerrado hasta que necesites abrirlo.

102. CORS (Cross-Origin Resource Sharing)
Cero: Una regla estricta de los navegadores web que bloquea a las páginas que intentan enviar o robar datos hacia otros dominios.

Wozniak: Un mecanismo de seguridad basado en cabeceras HTTP. Le dice al navegador si la página A.com tiene permiso para hacer un POST Request a la API de B.com.

Jobs: El terror de los novatos. Si conectas el formulario de tu Landing Page a un Webhook y no hace nada, a menudo es un error de CORS. Saber arreglar esto en minutos justifica tu tarifa por hora como consultor.

Objetivo: Saber que el error CORS se arregla en los "Headers" de la respuesta del servidor, no llorando sobre el código.

103. Environment Variables (Secretos de Producción)
Cero: Las cajas fuertes reales donde guardas contraseñas en el servidor de verdad.

Wozniak: Datos dinámicos inyectados en el contenedor Docker en el momento de la ejecución (vía archivos .env). Protegen tokens y passwords de la base de datos fuera del sistema de control de versiones (Git).

Jobs: Nivel de seguridad corporativo. El código de tus flujos puede ser visto por tus futuros empleados, pero las contraseñas reales que mueven dinero solo las conocen el servidor y tú.

Objetivo: Nunca, jamás, "quemar" (hardcodear) una API Key escribiéndola directamente en un módulo de Make si lo vas a exportar.

Wozniak: Automatización de la tubería de desarrollo (Pipeline). Herramientas como GitHub Actions ejecutan pruebas automáticamente cada vez que haces un Push, y si no hay errores, despliegan el código en el VPS.

Jobs: Velocidad de entrega insuperable. Te permite lanzar mejoras diarias al cliente sin tener que entrar al servidor manualmente. Cero esfuerzo físico, máximo valor percibido.

105. Disaster Recovery (Plan de Recuperación)

Wozniak: Extracción automatizada (mediante cron jobs) de las bases de datos y exportación de todos los archivos JSON (Blueprints) hacia un almacenamiento físico externo en frío.

Jobs: Tu responsabilidad máxima frente al cliente. Existen dos tipos de expertos: los que ya perdieron los datos de un cliente y los que van a perderlos. Vender un sistema corporativo sin un plan de contingencia es negligencia profesional.

Objetivo: Implementar la regla 3-2-1: Tres copias de tus datos, en dos medios distintos, una de ellas fuera de tu oficina/servidor principal.

116. Product-Market Fit (Ajuste Producto-Mercado)
Cero: Ese momento mágico de un negocio donde la gente te compra el servicio tan rápido que no das abasto para instalarlos.

Wozniak: Cuando el resultado de tus flujos de automatización resuelve una ineficiencia tan crítica y universal en un sector entero, que la fricción de adopción tecnológica cae a cero.

Jobs: El acelerador de la riqueza. Significa que no tienes que convencer a nadie; el mercado está desesperado por lo que tú tienes. Ocurre cuando dejas de adivinar qué quieren y escuchas sus problemas reales.

Objetivo: Buscar patrones. Si 3 clientes distintos de bienes raíces te piden el mismo CRM automatizado, ahí tienes un producto de software escalable.

117. Onboarding (Integración del Cliente)
Cero: Los primeros pasos que da un cliente justo después de firmar el contrato y pagarte.

Wozniak: Un flujo de automatización interno que recolecta credenciales (APIs, accesos), despliega la infraestructura en tu VPS y envía notificaciones de inicio de proyecto al cliente de forma autónoma.

Jobs: La confirmación de que tomó la decisión correcta. Un inicio caótico genera arrepentimiento y peticiones de reembolso. Un Onboarding suave, automático y profesional justifica tu precio elevado desde el primer minuto.

Objetivo: Automatizar un email de "Bienvenida y Próximos Pasos" para que se envíe en el instante en que el pago del cliente entra a Stripe.

118. Offboarding (Salida del Cliente)
Cero: El proceso ordenado y profesional cuando un cliente deja de trabajar contigo o se termina el contrato.

Wozniak: Revocación estricta de tokens de acceso (PATs, API Keys), destrucción de contenedores dedicados, exportación de bases de datos y entrega de código fuente o Blueprints (solo si aplica legalmente).

Jobs: Elegancia que trae referidos. Terminar bien una relación garantiza que te recomienden en el futuro. Además, proteger tu propia infraestructura retirando sus accesos evita vulnerabilidades de seguridad a tu agencia.

Objetivo: Nunca entregar la propiedad del ecosistema central o de las cuentas de administrador hasta que se haya cobrado la última factura.

119. Scope Creep (Corrupción del Alcance)
Cero: Cuando el cliente empieza a pedirte "una cosita más" gratis cada día, y terminas trabajando el doble por el mismo dinero pactado.

Wozniak: Alteración no documentada de los parámetros del proyecto original que requiere reestructuración del mapeo de datos, lógicas de ruteo adicionales y aumento del consumo de operaciones imprevistas.

Jobs: El destructor de rentabilidad de los programadores novatos. Un Arquitecto delimita el proyecto en el contrato. Si el cliente pide una nueva función, la respuesta es inmediata: "Claro que es posible. Eso requiere una reestructuración y un presupuesto adicional de 400€. ¿Avanzamos con la orden de compra?".

Objetivo: Acostumbrarte a redactar un documento que especifique exactamente qué SÍ hace la automatización y, más importante, qué NO hace.

120. SLA (Service Level Agreement - Acuerdo de Nivel de Servicio)
Cero: La promesa oficial por escrito de qué tan rápido vas a arreglar las cosas si se rompen.

Wozniak: Parámetros técnicos garantizados por contrato, como un Uptime del 99.9% de los servidores, latencia máxima aceptable, y tiempo de respuesta ante caídas de la API.

Jobs: El escudo legal y comercial. Justifica el cobro de un gran Retainer mensual. Si prometes darle soporte técnico a tu cliente en menos de 2 horas (SLA muy alto), puedes cobrarle el triple de mantenimiento que un novato que responde en 3 días.

Objetivo: Definir desde el día 1 que tu tiempo de respuesta estándar será de 24/48 horas, salvo que el cliente decida pagar la suscripción del paquete "Premium".
